# 先定目标

**禁止在这里修改代码**

--目标未定（设计全新方案、创造性工作）-->目标建立
--用户自己目标清晰，但你并不清楚他的目标（用户没说清楚 or 没展开）-->目标对齐
--目标清晰（达成某个确定的预期行为、执行具体行动）-->跳过

当你搞清楚目前状态是以上三个的哪一个时，end task 根据成事心法处理问题 / 处理用户消息，add & start task 对应的下一项。

**目标建立**：
这一部分的主要任务是，通过问问题确定目标。

问什么：

- 比如用户要做一个假期计划软件
- 先问终极目的：方便自己做计划
- 再问受众：用户自己
- 有没有人做过？（TickTick、日历）如果没有或者失败了，分析为什么失败（结构性问题）；如果成功了，分析这个成熟的竞品（功能强大，但是要付钱；电脑端不是一等公民）
- 分析这个产品的体验维度，询问用户差异化在哪里（那个爆点是什么）：免费、可控、简单、电脑端
- 再提一些具体方面的问题，比如：
  - 参考用途，建议可以加个番茄钟：用户同意
  - 日视图、周视图、月视图、事件试图都要有吗：只要日月
  - 要不要内置模版和时间统计：要
  - 怎么实现：Next.js，本地服务器
  - 要手机端访问吗：要，用 Tailscale

原则：

1. 一次只问一个问题，根据前面的回答构思问题
2. 可以先用工具去获取信息再问问题
3. 100 个问题比 0 个问题好
4. 尽量每个问题提供至少三个不同方面的引导性选项

当你至少提了 3 个问题，并获取了足够多的信息时，进入目标对齐环节

**目标对齐**：

这一部分的主要任务是，通过完整陈述你自己的目标，让用户就不同的地方来跟你讨论，来对齐你和用户的目标。

怎么做：

- 先审视现有的、你认为的目标，和所有用户消息
- 根据用户消息修改它，保持它和用户描述没有抵触。如果有两条用户消息态度相反（比如参考了你的反驳），参考最新的那条
- 思考：有更好的方案吗？这样做有没有什么坏处？这样做符合终极目标吗？
- 输出你理解的用户目标，和你的思考

组织格式：

例如对于做一个产品而言：

```
# 标题

**终极目标**：是什么，一句话

## 产品形态

**爆点**：产品往哪里搞。列几个词语（比如成本、可靠、透明、性能）

[在什么平台用]

[用什么方法做出来]

[和已有产品的差异化]

## 功能

详细描述产品提供的每项功能，精确到总结级别，例如：

### 番茄钟

- 可以关联事件
- 自定义专注和休息时间
- 支持暂停
- 持久化记录
```

例如对于修 bug：

特别要说明的是，修 bug 时，禁止在目标对齐这个环节就去调查根因。

```
# 标题

**现有行为**：

**期望行为**：

**猜测原因**：

- 原因 1
- 原因 2
```
